## 🔹 Présentation du Projet

**Projet MMO Unity** est un jeu vidéo de style RPG en 2D. Ce projet a été développé pour offrir des mécaniques de combat exigeantes et a été réalisé dans un cadre pédagogique.

* **Contexte** : Projet de 2e année au sein du Département Informatique (Développement Jeu Vidéo) de l'Université Paris Nanterre, achevé en Mai 2025.
* **Objectifs** : Développer un jeu fluide avec un système de combat dynamique, approfondir la programmation orientée objet en C#, et maîtriser l'architecture du moteur Unity.
* **Fonctionnalités Clés** : 
    * **Inventaire Actif** : Système de slots visibles à l'écran permettant de basculer rapidement entre différentes armes.
    * **Système de Combat** : Armes avec comportements distincts (arc lançant des flèches, bâton générant un laser magique dynamique).
    * **Contrôles Modernes** : Utilisation du Unity Input System pour une gestion multiplateforme des entrées (clavier, souris, manette).
    * **Interface & UX** : HUD synchronisé en temps réel avec l'inventaire et transitions visuelles douces via un système de fondu (UIFade).

###

## 🔹 Installation & Déploiement

1.  **Cloner le projet** : Récupérer le code source via le dépôt Git.
2.  **Ouvrir avec Unity** : Lancer Unity Hub et ouvrir le dossier du projet. Assurez-vous d'avoir installé les packages standards (notamment l'Input System).
3.  **Lancement de la scène** : Dans l'éditeur, naviguer vers les scènes et ouvrir la scène principale (ex: `Scene1`).
4.  **Exécution** : Cliquer sur le bouton "Play" pour tester les mécaniques de déplacement et de combat directement dans l'éditeur.

###

## 🔹 Structure de l'Architecture

L'architecture du code est modulaire et repose sur la séparation des responsabilités :

* **PlayerController** : Gestion des mouvements, de l'orientation vers la souris et des interactions.
* **ActiveInventory** : Cœur du système de slots, gérant la rotation et l'instanciation des armes équipées.
* **InputManager** : Centralisation des événements d'entrée via des *Action Maps*.
* **UIManager** : Façade mettant à jour l'affichage dynamique (nom de l'arme, icônes, animations de transition).
